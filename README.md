# Office-VLAN-Setup# 🌐 Office Network Design using VLAN

## 📌 Project Overview
This project focuses on designing and implementing a **VLAN-based office network** to improve performance, security, and network management.  
Different departments are separated using VLANs to ensure controlled and efficient communication.

---

## 🎯 Objectives
- Segment network into different departments  
- Improve security using VLAN isolation  
- Enable controlled communication using inter-VLAN routing  
- Optimize network performance  

---

## 🛠️ Tools & Technologies
- Cisco Packet Tracer  
- Networking Concepts (TCP/IP, VLAN, Subnetting, Routing)  

---

## 🏗️ Network Design
- Departments: HR, Sales, IT  
- Each department assigned to a separate VLAN  
- Switches used for VLAN configuration  
- Router used for inter-VLAN communication  
- DHCP configured for automatic IP assignment  

---

## 🔧 Configuration Details

### ✅ VLAN Setup
- VLAN 10 → HR  
- VLAN 20 → Sales  
- VLAN 30 → IT  

---

### ✅ IP Addressing
- Used subnetting to assign IP ranges for each VLAN  
- Example:
  - HR: 192.168.10.0/24  
  - Sales: 192.168.20.0/24  
  - IT: 192.168.30.0/24  

---

### ✅ Inter-VLAN Routing
- Configured router to allow communication between VLANs  
- Ensured controlled and secure access  

---

### ✅ DHCP Configuration
- Enabled DHCP for automatic IP assignment  
- Reduced manual configuration effort  

---

## ⚙️ Implementation Steps
1. Create VLANs on switches  
2. Assign switch ports to respective VLANs  
3. Configure trunk ports between switches  
4. Set up router for inter-VLAN routing  
5. Configure DHCP for each VLAN  
6. Test connectivity  

---

## 🧪 Testing & Validation
- Verified communication within VLANs  
- Tested inter-VLAN routing  
- Used ping command to check connectivity  

---

## 📊 Results
- Improved network organization  
- Enhanced security through segmentation  
- Efficient traffic management
